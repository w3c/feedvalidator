Message

Undefined media-range parameter

Explanation

Media type parameters are allowed within app:accept, but app:accept has no notion of preference - "accept-params" or "q" arguments, as specified in Section 14.1 of [RFC2616] are not significant.

Solution

Generally, it is best to remove the ;q=qvalue parameter from the media-range.
